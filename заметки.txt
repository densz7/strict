1. Использовать css reset для кроссбраузерности (для более корректного отображения страницы в разных брауезрах).
2. Для удобства можно использвать свойство css - box-sizing: border-box (но не обязательно).
3. Не нужно для одного элемента создавать несколько родительских блоков, а лучше для самого элемента задавать несколько классов:

	1) Если один и тот же элемент будет использоваться на странице несколько раз,
	то его лучше всего сделать блоком для дальнейшего перемещения по странице.
	Для этого необходимо создать дополнительный класс для элемента, который будет содержать
	в себе общие стили для всех элементов, повторяющихся на странице.
	
	Если же повторения элементов нет, то просто создаем класс, который будет содержать стили для оформления 
	внешнего вида элемента.

	2) Для элемента создается класс, который будет отвечать за его позиционирование 
	относительно родительского блока и других дочерних элементов.

	3) Если какой-то из повторяющихся элементов необходимо видоизменить 
	(например, поменять цвет текста, цвет фона и т.д.), 
	то элементу необходимо добавить дополнительный класс в виде модификатора. 

Итого: 
	1) Элемент не повторяется: создаем 2 класса - один для внешнего вида, второй для позиционирования.

	2) Элемент будет повторяться: создаем 3 класса - один для общего внешнего вида(дефолтного), 
	второй для позиционирования относительно родителя и третий(модификатор) для изменения внешнего вида.

4. При копировании какого-либо элемента в другое место на странице меняются стили (если это необходимо) класса, описанного в пункте 3.2
